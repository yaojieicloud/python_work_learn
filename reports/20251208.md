# 📝 今日作业

## 作业需求：扩展 **学生信息管理系统**

### 新增功能要求

1. **文件操作**

   - 将学生信息保存到本地文件（如 `students.txt`）。
   - 支持从文件中读取学生信息并加载到系统。
   - 使用 `with open()` 保证文件安全关闭。
   - 支持目录检查与自动创建（如 `data/` 文件夹）。
2. **数据库操作**

   - 使用 `sqlite3` 创建一个 `students` 表，包含字段：姓名、学号、成绩。
   - 将新增的学生信息同步写入数据库。
   - 支持从数据库查询学生信息。
   - 支持更新与删除数据库中的学生记录。
3. **综合要求**

   - 保持原有功能（增删查改、正则校验、平均成绩统计）。
   - 新增方法：
     - `save_to_file()` → 将学生信息保存到文件。
     - `load_from_file()` → 从文件加载学生信息。
     - `save_to_db()` → 将学生信息保存到数据库。
     - `query_from_db()` → 从数据库查询学生信息。

### 作业目标

- 综合运用 **文件操作与数据库操作**，实现数据的持久化与结构化管理。
- 巩固面向对象思想，提升系统的扩展性。
- 为后续学习 ORM 框架打下基础。

### 提示

- 文件存储适合快速保存与加载。
- 数据库存储适合复杂查询与多用户场景。
- 可以在 `StudentManager` 中同时支持两种存储方式，形成一个 **双存储系统**。
